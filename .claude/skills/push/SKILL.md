# Push Skill

現在のブランチをGitHubリモートリポジトリへプッシュする。

## 手順

1. 以下のコマンドを並列実行し、現在の状態を確認する
   - `git status`: 未コミットの変更がないか確認
   - `git branch -vv`: 現在のブランチとリモート追跡状態を確認
   - `git log @{upstream}..HEAD --oneline 2>/dev/null || echo "no upstream"`: プッシュ対象のコミットを確認
2. 未コミットの変更がある場合はユーザーに警告し、先にコミットするか確認する
3. リモート追跡ブランチの有無を判定する
   - **追跡ブランチなし**: `git push -u origin <branch>` で初回プッシュ
   - **追跡ブランチあり**: `git push` で通常プッシュ
4. プッシュ結果を確認し、成功・失敗をユーザーに報告する

## 注意事項

- `--force` / `--force-with-lease` / `-f` は一切使用しない。ユーザーが要求しても拒否する
- プッシュ先がリジェクトされた場合は、原因を説明しユーザーに対処方法（pull・rebase等）を提示する
- プッシュ前に未コミットの変更がある場合は必ずユーザーに確認を取る
